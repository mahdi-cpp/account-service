Account Service API

Create Read Update Delete Auth

This is an interactive guide to the Golang Account Service REST API. It outlines the primary operations, requests, and expected responses. Click on any scenario to expand its details.

Create a New Account

This section details how to create new user accounts. It covers the successful creation flow and potential errors like submitting data for an account that already exists.

POST /api/v1/accounts

Request Body

{
  "email": "user@example.com",
  "password": "securepassword123"
}

Success Response (201 Created)

Headers:
Location: /api/v1/accounts/{id}

Body:
{
  "id": "generated-uuid-1234"
}

Request Body

{
  "email": "existing@example.com",
  "password": "anotherpassword"
}

Error Response (409 Conflict)

{
  "error": "Email address already in use."
}

Retrieve an Existing Account

This endpoint allows for fetching the details of a single account using its unique ID. Note that sensitive information like the password is not returned.

GET /api/v1/accounts/{id}

Request

Path: /api/v1/accounts/valid-id-123

Success Response (200 OK)

{
  "id": "valid-id-123",
  "email": "user@example.com",
  "createdAt": "2025-08-31T17:33:00Z"
}

Request

Path: /api/v1/accounts/non-existent-id

Error Response (404 Not Found)

{
  "error": "Account not found."
}

Update an Account

Use this endpoint to modify details of an existing account. The request body should contain only the fields that need to be updated.

PUT /api/v1/accounts/{id}

Request Body

{
  "email": "new.email@example.com"
}

Success Response (200 OK)

{
  "id": "valid-id-123",
  "email": "new.email@example.com"
}

Request Body

{
  "email": "not-an-email"
}

Error Response (400 Bad Request)

{
  "error": "Invalid email format."
}

Delete an Account

This operation permanently deletes an account. This action is irreversible. A successful deletion returns no content.

DELETE /api/v1/accounts/{id}

Request

Path: /api/v1/accounts/valid-id-123

Success Response (204 No Content)

(Empty Response Body)

Request

Path: /api/v1/accounts/non-existent-id

Error Response (404 Not Found)

{
  "error": "Account not found."
}

User Authentication

Handles user login by validating credentials and returning an authentication token for subsequent authenticated requests.

POST /api/v1/auth/login

Request Body

{
  "email": "user@example.com",
  "password": "securepassword123"
}

Success Response (200 OK)

{
  "token": "jwt-auth-token-string"
}

Request Body

{
  "email": "user@example.com",
  "password": "wrongpassword"
}

Error Response (401 Unauthorized)

{
  "error": "Invalid credentials."
}
